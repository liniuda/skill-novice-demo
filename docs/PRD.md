## 一、产品概述与愿景

### 1.1 产品定位

**智服工坊 X（CCO Skill Studio）** 是面向 CCO 智能客服体系的一站式 AI Skill 配置、运营与市场化平台。它让不具备编码能力的**运营同学**能够通过可视化界面完成 AI Agent 技能的创建、调优、发布与效果追踪，最终驱动 cco-ai-service 对话引擎的智能化升级。

### 1.2 产品愿景

> 让每一个运营同学都能成为 AI 训练师——零代码配出高质量 Skill，用数据验证效果，用市场沉淀最佳实践。

### 1.3 核心价值主张

| 维度 | 价值 |
|------|------|
| 降本 | 运营自助配置，减少对算法/工程的依赖 |
| 提效 | AI 起草 + 模板 fork，Skill 上线时间从天级缩短到小时级 |
| 质量 | 冲突检测 + 归因分析，数据驱动持续优化 |
| 沉淀 | Market 社区化，最佳实践跨团队复用 |

## 二、目标用户与使用场景

### 2.1 用户画像

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| AI 训练师（运营） | CCO 各 BU 的智能客服运营同学 | 快速配出有效 Skill、监控效果、持续迭代 |
| 产品经理 | 智能客服产品线 PM | 跨空间看全局数据、管理发布策略 |
| 算法工程师 | cco-ai-service 维护者 | 查看拟人化 YAML 产物、调试 MCP 连接 |
| 团队管理者 | BU Leader | 看团队 KPI 达成、审核高风险发布 |

### 2.2 核心使用场景

1. **场景 A：新建并发布一个退款 Skill**  
   运营同学进入智服工坊 → 选择「商家小蜜」空间 → 用 AI 起草描述业务规则 → 系统解析为结构化 Skill → 绑定 MCP 工具 → 发布前冲突检测 → 灰度发布上线

2. **场景 B：效果下降时快速定位**  
   收到报警 → 进入效果分析 → 查看场景归因 → 定位到「物流时效」场景解决率下降 → 点击 AI 建议补全话术 → 重新发布

3. **场景 C：复用市场优质 Skill**  
   新 BU 入驻 → 浏览 Skill 广场 → 找到官方「AI 客服路由」Skill → 一键 fork 到自己空间 → 微调后发布

4. **场景 D：配置 AI 客服拟人化风格**  
   运营同学进入拟人化中心 → 配置语气/emoji/口头禅/兜底策略 → 预览 System Prompt → 导出 YAML → 下发到 cco-ai-service

## 九、数据模型概要

### 9.1 核心实体

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Space      │────<│    Skill     │────<│  SkillVersion│

└──────────────┘     └──────────────┘     └──────────────┘
                            │                      
                     ┌──────┴──────┐               
                     │             │               
              ┌──────────┐  ┌──────────┐          
              │ MCP Bind │  │ Eval Case│          
              │(工具绑定) │  │(评估用例) │          
              └──────────┘  └──────────┘          
                     
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
```

### 9.2 Skill 主表字段设计

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | UUID 主键 |
